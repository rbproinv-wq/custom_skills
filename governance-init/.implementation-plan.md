**Action:** Rewrite from 44 lines to full skill with standard sections.

**New structure:**
```markdown
---
name: governance-init

# Governance Init — Phase 0: Project Scaffolding

## When to Use This Skill

- Starting a new Crush-managed project
- Setting up governance structure for an existing project
- After cloning a repo that needs Crush config

## Do Not Use This Skill When

- The project already has a complete .crush/ structure
- You only need to add a single config file
- The task is about application code, not project setup

## Workflow

### Step 1: Wizard (Recommended)
    bash scripts/init.sh --wizard
    → Answers 7 questions, generates scaffold, validates

### Step 2: Detect (Existing Project)
    bash scripts/init.sh --detect
    → Auto-detects stack, asks for confirmation

### Step 3: Minimal (Manual config)
    bash scripts/init.sh
    → Generic scaffold, no questions

### Step 4: Validate Environment
    bash scripts/validate_env.sh
    → Checks runtimes, API keys, ports, git config

### Step 5: Post-Init Validation
    bash scripts/validate.sh
    → Verifies all artifacts are in place

## Edge Cases

- Empty directory → generic scaffold
- Existing project → detects stack, preserves existing files
- Missing git config → warns, suggests git config
- Port conflict → detects, suggests alternative
- Cancel mid-wizard → no files written

## Verification Checklist

Before declaring success, confirm:
- [ ] `.crush.md` exists and has stable + dynamic blocks
- [ ] `crush.json` exists and is valid JSON
- [ ] `.spec/` directory exists
- [ ] `.crush/prompts/` has at least refactor.md, review.md, test.md
- [ ] Git repository is initialized
- [ ] API keys are configured (validate_env.sh passes)
- [ ] LLM Router is running (if selected)

## Output Format

After successful init, present:
    @governance-init:
    ✅ Governance structure installed
    - Type: [generic | python-fastapi | typescript-next]
    - Files: 12 created in .crush/, .spec/, .git/
    - Next: run product-analyst-mvp for PRD
```

**Verify:**
- ✅ SKILL.md has all 8 standard sections
- ✅ Output format section guides the LLM behavior
- ✅ Edge cases covered
- ✅ No hardcoded `~/.agents/scripts/` paths — use relative `scripts/` paths
