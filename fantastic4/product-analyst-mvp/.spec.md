# .spec.md — product-analyst-mvp: Structured PRD Engine with Discovery Framework

## 1. Objective

Refine `product-analyst-mvp` from a solid but manual PRD skill into a **structured, framework-driven product discovery engine** with triage, sourced research, mandatory non-goals, RICE prioritization, and multi-template PRD output. Integrate patterns from `idea-os`, `product-manager-toolkit`, `saas-mvp-launcher`, and `product-manager`.

## 2. Key Changes

| Current | Future |
|---------|--------|
| Single PRD template (11 sections) | 4-tier PRD templates (Standard, One-Page, Agile Epic, Feature Brief) + auto-selection by project type |
| Basic cost estimation | RICE prioritization + Effort mapping (XL/L/M/S/XS person-months) |
| Optional research mention | Mandatory triage + sourced research before PRD (mínimo 3 web searches, flag assumptions as `[assumption]`) |
| No non-goals section | **Mandatory** non-goals per MVP iteration (kill criteria) |
| Basic verification checklist | Post-PRD reader test (fresh agent validation) |
| Hardcoded "Skill 2/3/4" references | Pipeline-relative references (from frontmatter) |
| 1 product discovery framework | Library of 5+ frameworks with auto-selection by project type |

## 5. New Workflow (9 Steps)

```
Step 0: Triage — Classify request (Tier × Sophistication)
Step 1: Analyze Input & Extract — (existing)
Step 2: Research — 3+ web searches, flag assumptions
Step 3: Apply Framework — Auto-select from 5+ frameworks
Step 4: RICE Prioritization — Score features
Step 5: Define MVP Scope — (existing) + pre-pay gate: "3 pagariam?"
Step 6: Create Personas — (existing) + JTBD per persona
Step 7: Generate PRD — Template auto-selected from 4 options
Step 8: Reader Test — Fresh agent validates
Step 9: Handoff — Output + artifacts
```

## 6. Edge Cases

| Scenario | Handling |
|----------|----------|
| User refuses research | Document as `[assumption]` in all affected sections, proceed with PRD |
| User says "no competitors" | Research anyway — if truly none, note "blue ocean" with supporting evidence |
| PRD > 200 lines | Default to One-Page template (limits scope), full details in references |
| Idea Tier T3 (regulated/marketplace) | Auto-select Standard template, add compliance/legal sections |
| All features score low on RICE | Warn user, suggest pivot or kill criteria before proceeding |

## 7. Acceptance Criteria

1. PRD output has 14 sections (Standard) or correct sections for chosen template
2. Non-goals section is always present with ≥3 entries
3. `[assumption]` flags appear next to unsourced claims
4. RICE scores present for all MVP features
5. Reader test passes (fresh agent can explain the PRD correctly)
6. All references to pipeline skills use pipeline-relative names (no hardcoded numbers)
7. Pre-pay gate question asked before MVP scope finalized

## 8. Non-Goals

- Not implementing the research scripts (web_search tool assumed available)
- Not generating code or technical specifications
- Not persisting PRD across sessions (output is .spec/prd.md file)
